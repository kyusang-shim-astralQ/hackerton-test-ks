# WORKPLAN — 팀 분업 가이드 (2시간 해커톤)

> 모노레포 + 백지 MVP 기준. 핵심 원칙: **한 사람 = 한 기능을 풀스택(FE+BE)으로 소유**, **계약(`docs/`)이 단일 소스**, **파운데이션 먼저**.

## 0. 큰 그림
- **파운데이션(공유)** 을 1~2명이 먼저 깐다 → 나머지는 각자 기능을 **병렬**로.
- 각 기능은 자기 폴더만 건드림(`backend/app/features/<x>/` + `frontend/features/<x>/` + 자기 store 슬라이스/ i18n 파일) → **충돌 0**.
- 계약(api.md/data-models/design-system)만 지키면 **자동으로 맞물림**. 상위 기능은 목/시드로 안 기다림.

## 1. 역할 분담 (예: 4명 — 팀 크기에 맞게 조정)
| 사람 | 0:00–0:30 (파운데이션) | 0:30–1:30 (기능, FE+BE 풀스택) |
|---|---|---|
| **A (리드)** | git/모노레포 골격 + `CLAUDE.md` + **BE 파운데이션**(`be-01`) | **f1-structure**(ASE 파싱·3D) + 통합/데모 총괄 |
| **B** | **FE 파운데이션**(`01-foundation`) | **f2-plan**(AI 플랜 — 데모 핵심) + f3-inp |
| **C** | (B 도와 FE 셸 검증) | **f4-jobs**(모니터링 대시보드·목 스트림) |
| **D** | (A 도와 BE 스키마/계약) | **f5-report**(AI 리포트 — 데모 핵심) + (여유 시 f6) |

- **3명**: 파운데이션 A(BE)·B(FE) 동시, 이후 A=f1+f3, B=f2+f5, C=f4(+f6). f6은 버려도 됨.
- **5~6명**: 1기능=1명으로 쪼개고 1명은 "통합/데모" 전담.
- 배분 기준: **데모 핵심(f2 플랜·f5 리포트)** 에 LLM 능숙한 사람, f4는 UI 비중 큼.

## 2. 2시간 타임라인
| 시각 | 할 일 |
|---|---|
| **0:00–0:15** | repo clone, `.env`에 **공유 `CLAUDE_API_KEY`**, 역할 확정. FE/BE **파운데이션 프롬프트** 가동(`01-foundation`, `be-01-foundation`). |
| **0:15–0:30** | 파운데이션 착지 확인: FE `npm run dev`(:3000) 뜸, BE `/health`(:8000) OK. 나머지는 자기 기능 폴더 스캐폴드 + 스펙 정독. |
| **0:30–1:30** | **기능 병렬 구현.** 각자 `be-0X`(백엔드) + `0X`(프런트) 프롬프트 실행. **MOCK 모드 ON**(`NEXT_PUBLIC_MOCK=1`)로 단독 동작. |
| **1:30–1:50** | **통합.** 브랜치 머지 → end-to-end(f1→f2→f3→f4→f5) 스모크 테스트. 키 있는 실동작(f1/f2/f3/f5)은 **MOCK 끄고** 실연결. |
| **1:50–2:00** | 데모 리허설 + 버그 픽스. 데모 경로 고정(아래 §5). |

## 3. git 워크플로
- `main` = 파운데이션(A가 초기 푸시). 각자 `feature/fX` 브랜치(또는 worktree)에서 작업 → 자주 커밋 → 1:30에 머지.
- 파운데이션이 **공유 표면(store/steps/i18n/api base)을 전부 소유**하므로 기능 브랜치 간 충돌이 거의 없음(01-foundation "병렬 안전" 절).
- 소규모면 트렁크 기반도 OK — 단 "자기 폴더만 건드린다" 규율 필수.

## 4. 통합 체크포인트 (자동으로 맞물리게 하는 규약)
- **계약 준수 = 인터롭**: 각 기능은 자기 `api.md` 입출력·`data-models.md` 필드를 그대로. 바꾸려면 계약(공유 파일)부터 합의.
- **store 슬라이스 규약**: 자기 기능 상태만 자기 슬라이스에. SummaryPanel/진행률은 파운데이션 규약대로 구독.
- **1:30 스모크 테스트**: 데모 메인 경로가 끝까지 흐르는지 한 번 통과시키고 거기서부터 다듬기.

## 5. 데모 시나리오 (심사에서 보일 것)
**메인(진짜 도는 흐름):** CIF 업로드 → **AI가 멀티스텝 플랜 설계(f2, 실제 LLM)** → INP 생성(f3) → (f4 목 실행 모니터링) → **AI 리포트(f5, 실제 LLM)**.
→ "구조만 올리면 AI가 계산을 계획·실행·해석한다"가 한 화면 흐름으로 보임. **f2·f5의 LLM 출력이 하이라이트.**
보조(시간 되면): f6 12-레벨 벤치마크(목).

## 6. 리스크 & 팁
- **API 키**: 시작하자마자 공유, 레이트리밋 대비(플랜/리포트 호출 집중).
- **MOCK 기본 ON**: 클러스터 없이도 전 흐름이 돌게. 통합 때 f1/f2/f3/f5만 실연결.
- **시간 부족 시 우선순위**: f6 → f4 디테일 순으로 줄이고, **f1→f2→f3→f5 척추는 사수**.
- **마지막 30분은 통합/데모 1명 전담** — 새 기능 추가 금지, 데모 경로 안정화만.
- 막히면 그 지점이 **스펙(계약)의 빈틈**일 확률이 큼 → 빠르게 계약을 보강하고 양쪽 동기화.
